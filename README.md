# OpenClaude

> The open source operating system for AI-powered businesses.

---

## What It Is

OpenClaude is a multi-agent workspace built on top of Claude Code. It turns a single AI assistant into a team of 9 specialized agents -- each with its own domain, skills, persistent memory, and automated routines. The result is a production system that runs daily operations for a founder/CEO: from morning briefings to financial reports, community monitoring, social analytics, and end-of-day consolidation.

**This is not a chatbot.** It is a real operating layer that runs routines, generates HTML reports, syncs meetings, triages emails, monitors community health, tracks financial metrics, and consolidates everything into a unified dashboard -- all automated.

---

## Key Features

- **9 Specialized Agents** -- Clawdia (ops hub), Flux (finance), Atlas (projects), Kai (personal/health), Pulse (community), Sage (strategy), Pixel (social media), Nex (sales), Mentor (courses)
- **126 Skills** -- organized by domain prefix (`evo-`, `social-`, `fin-`, `int-`, `prod-`, `mkt-`, `gog-`, `obs-`, `discord-`, `pulse-`, `sage-`)
- **27 Automated Routines** -- daily, weekly, and monthly ADWs (AI Developer Workflows) managed by a scheduler
- **18 Integrations** -- Google Calendar, Gmail, Linear, GitHub, Discord, WhatsApp, Telegram, Stripe, Omie, Fathom, Todoist, YouTube, Instagram, LinkedIn, and more
- **17 HTML Report Templates** -- dark-themed dashboards for every domain (finance, community, social, licensing, strategy, health)
- **Persistent Memory** -- two-tier system (CLAUDE.md hot cache + memory/ directory) so the AI remembers context across sessions
- **Full Observability** -- JSONL logs, execution metrics, cost tracking per routine

## Architecture

```
User (human)
    |
    v
Claude Code (orchestrator)
    |
    +-- Clawdia   -- Ops hub (agenda, emails, tasks, decisions, dashboard)
    +-- Flux      -- Finance (Stripe, ERP, MRR, cash flow, monthly close)
    +-- Atlas     -- Projects (Linear, GitHub, licensing, milestones)
    +-- Kai       -- Personal (health, habits, routine -- isolated domain)
    +-- Pulse     -- Community (Discord, WhatsApp, sentiment, FAQ)
    +-- Sage      -- Strategy (OKRs, roadmap, prioritization, scenarios)
    +-- Pixel     -- Social media (content, calendar, cross-platform analytics)
    +-- Nex       -- Sales (pipeline, proposals, qualification)
    +-- Mentor    -- Courses (learning paths, modules, academy)
```

Each agent has a dedicated system prompt (`.claude/agents/`), a slash command (`/clawdia`, `/flux`, etc.), and persistent memory (`.claude/agent-memory/`).

## Quick Start

```bash
# 1. Clone
git clone https://github.com/EvolutionAPI/open-claude.git
cd open-claude

# 2. Setup wizard (creates config, .env, CLAUDE.md, folders)
make setup

# 3. Edit .env with your API keys
nano .env

# 4. Start the web dashboard
make dashboard-app
# Open http://localhost:8080 — first run shows setup wizard

# 5. Start automated routines
make scheduler

# 6. See all available commands
make help
```

### Web Dashboard (OpenClaude)

A full web UI with:
- **Overview** — unified dashboard with metrics from all agents
- **Reports** — HTML reports generated by routines
- **Agents, Skills, Routines** — browse and manage the framework
- **Chat** — embedded Claude Code terminal (xterm.js + WebSocket)
- **Services** — start/stop scheduler, Telegram bot, Docker containers
- **Auth** — login, roles, permissions, audit log
- **Setup Wizard** — guided first-run configuration via web

```bash
make dashboard-app   # Start Flask + React on :8080
```

## Workspace Structure

```
OpenClaude/
+-- .claude/
|   +-- agents/          -- 9 agents with system prompts
|   +-- commands/        -- 9 slash commands
|   +-- skills/          -- 126 skills organized by prefix
|   +-- templates/       -- 17 HTML + 11 Markdown templates
|   +-- agent-memory/    -- persistent memory per agent
+-- ADWs/
|   +-- runner.py        -- core runner (Rich output + logs + Telegram)
|   +-- rotinas/         -- 27 ADW scripts (Python)
|   +-- logs/            -- JSONL + detailed logs + metrics
+-- dashboard/
|   +-- backend/         -- Flask API + SQLAlchemy + WebSocket
|   +-- frontend/        -- React + TypeScript + Tailwind
+-- social-auth/         -- OAuth app for social networks
+-- config/              -- workspace.yaml, routines.yaml
+-- _evo/                -- Evo Method (development framework)
+-- memory/              -- global persistent memory (people, projects, glossary, trends)
+-- scheduler.py         -- automated routine scheduler
+-- Makefile             -- 44+ make targets
+-- Dockerfile           -- Docker image for VPS deployment
+-- docker-compose.yml   -- containerized scheduler + telegram
+-- CLAUDE.md            -- global Claude context
+-- ROTINAS.md           -- complete routine guide
+-- ROADMAP.md           -- improvement plan
```

## Observability

```bash
make logs          # Latest JSONL entries
make logs-detail   # List detailed logs
make logs-tail     # Latest full log
make metrics       # Accumulated metrics per routine (runs, success rate, time, cost)
make clean-logs    # Remove logs older than 30 days
```

## Docker Deployment

```bash
make docker-build     # Build image
make docker-up        # Start scheduler + telegram
make docker-down      # Stop containers
make docker-logs      # Real-time logs
make docker-run ADW=good_morning.py  # Run a routine manually
```
